# Persona Mapping

> Map stakeholders and personas using Power-Interest matrices, RACI charts, and influence analysis for effective engagement strategies.

## Overview

Knowing who your users are (persona definition) is only half the picture. Persona Mapping answers the follow-up questions: which stakeholders have the power to block or accelerate your project? Who needs to be consulted vs. merely informed? How should you engage each group given their influence, interest, and attitude?

This skill provides three complementary frameworks for stakeholder analysis. The Power-Interest Matrix segments stakeholders into four quadrants (manage closely, keep satisfied, keep informed, monitor) based on their power and interest levels. The RACI Matrix assigns clear accountability for each task or decision. The Salience Model adds urgency and legitimacy dimensions for more nuanced prioritization.

Within the SkillStack collection, Persona Mapping is the stakeholder strategy skill. It takes the personas defined by the Persona Definition skill and places them in organizational context, determining engagement cadence, communication channels, and key messages for each stakeholder group.

## Usage Examples

**Map stakeholders for a platform migration:**
```
We're migrating from a legacy monolith to microservices. Map all stakeholders (engineering leads, product managers, QA team, executives, end users) into a Power-Interest Matrix. Define an engagement strategy for each quadrant.
```

**Create a RACI chart for a product launch:**
```
Build a RACI matrix for our product launch. Tasks include: feature development, QA testing, documentation, marketing materials, infrastructure setup, and go/no-go decision. Stakeholders are: PM, engineering lead, QA lead, tech writer, marketing, and VP of Engineering.
```

**Analyze stakeholder resistance:**
```
Our new testing framework adoption is meeting resistance from some teams. Create stakeholder profiles for each team lead. Identify their influence level, current attitude, and design a tailored engagement strategy to move resistors toward neutral or supportive.
```

**Prioritize stakeholders using the Salience Model:**
```
We have 12 stakeholders for our API redesign. Use the Salience Model to classify each by power, legitimacy, and urgency. Identify the definitive stakeholders who need the most attention and the latent ones we can monitor passively.
```

**Design a communication plan:**
```
Based on our stakeholder map, create a communication plan. For each stakeholder group, specify the engagement frequency, preferred channel, key messages to emphasize, and what level of detail they need.
```

## Quick Start

1. **List all stakeholders** -- Include anyone who influences, is affected by, or has interest in your project.

2. **Assess Power and Interest** -- Rate each stakeholder's power (ability to affect outcomes) and interest (how much they care) as High or Low.

3. **Plot the Power-Interest Matrix** -- Place each stakeholder in the appropriate quadrant: Manage Closely (high power, high interest), Keep Satisfied (high power, low interest), Keep Informed (low power, high interest), or Monitor (low power, low interest).

4. **Assign RACI roles** -- For each key task or decision, assign exactly one Accountable person, one or more Responsible people, and appropriate Consulted/Informed stakeholders.

5. **Create engagement strategies** -- For each stakeholder or group, define the engagement frequency, channel, and key messages using the stakeholder template.

6. **Review and update** -- Stakeholder dynamics change. Revisit your map at major project milestones.
