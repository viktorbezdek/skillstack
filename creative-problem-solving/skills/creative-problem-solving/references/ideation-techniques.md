# Ideation Techniques

## Divergent Thinking Methods

### SCAMPER
Systematic modification prompts.
- **S**ubstitute: What can be replaced?
- **C**ombine: What can be merged?
- **A**dapt: What can be adjusted?
- **M**odify: What can be changed in scale/form?
- **P**ut to other use: What's another application?
- **E**liminate: What can be removed?
- **R**everse/Rearrange: What if we flipped it?

### Morphological Analysis
Systematically combine attributes.
1. Break problem into dimensions/attributes
2. List options for each dimension
3. Combine across dimensions
4. Evaluate combinations

Example: Product delivery
- Speed: Instant, Same-day, Next-day
- Method: Drone, Vehicle, Pickup
- Cost: Premium, Standard, Budget
= 3×3×3 = 27 combinations to explore

### Attribute Listing
Vary properties independently.
- List all attributes of object/process
- Modify each attribute: Increase, decrease, invert
- "What if this was bigger/smaller/faster/slower?"

### Forced Connections
Link unrelated concepts.
- Random word → "How does [word] relate to our problem?"
- Combine disparate industries/fields
- "What's the [Industry A] solution to [Industry B] problem?"

## Analogical Thinking

### Biomimicry
Learn from nature.
- How do organisms solve similar problems?
- Evolution has tested solutions for millions of years
- Examples: Velcro (burrs), bullet trains (kingfisher)

### Cross-Domain Transfer
Import solutions from other fields.
- Manufacturing → Software (lean, kanban)
- Medicine → Business (triage, diagnosis)
- Sports → Work (coaching, deliberate practice)

### Synectics
Make unfamiliar familiar, familiar unfamiliar.
- **Direct analogy**: "What's this like?"
- **Personal analogy**: "What if I were this thing?"
- **Symbolic analogy**: Metaphors and poetry
- **Fantasy analogy**: Magic/impossible solutions → inspire real ones

## Scenario Exploration

### Future Scenarios
Develop multiple plausible futures.
- Identify key uncertainties
- Create 2×2 matrix of high-impact uncertainties
- Develop narrative for each quadrant
- Plan for multiple futures, not just expected one

### Premortem
Assume failure, work backwards.
- "It's 12 months from now and we failed. Why?"
- Uncover hidden risks and assumptions
- More psychologically safe than "What could go wrong?"

### Success Scenarios
Assume success, work backwards.
- "We achieved extraordinary results. How?"
- Identify what would need to be true
- Find blocking/enabling factors

## Combination & Synthesis

### Concept Blending
Merge distinct concepts into novel hybrid.
- Smartphone = Phone + Computer
- Ridesharing = Taxi + Social Network + Mobile Tech
- Look for: Structure from A + Properties from B

### Progressive Refinement
Build on ideas iteratively.
1. Generate raw ideas (quantity)
2. Cluster similar ideas
3. Combine/enhance clusters
4. Refine top candidates
5. Prototype/test

### Yes, And...
Improvisational building.
- Accept and build on ideas
- Defer judgment
- Create momentum
- "Yes, and we could also..."

## Provocation Techniques

### Escape Assumptions
Identify and negate core beliefs.
- "This product must be physical" → What if digital?
- "Customers buy from us" → What if we buy from them?
- Use to break conceptual fixedness

### Absurd Solutions
Intentionally impractical to spark practical.
- "What's the most ridiculous solution?"
- Extract kernel of insight
- "What if there were no rules/constraints?"

### Exaggeration
Push to extremes.
- "What if we had infinite budget/time/people?"
- "What if we had zero budget/time/people?"
- "What if this took 1 second? 10 years?"

## Systematic Innovation

### TRIZ (Theory of Inventive Problem Solving)
40 Inventive Principles (selection):
1. **Segmentation**: Divide into parts
2. **Taking out**: Extract problematic part
3. **Local quality**: Different areas, different functions
4. **Asymmetry**: Change from symmetric to asymmetric
5. **Merging**: Combine operations
6. **Universality**: Object performs multiple functions
7. **Nested doll**: Object contains another
8. **Intermediary**: Use intermediate object
9. **Preliminary action**: Do in advance
10. **Beforehand cushioning**: Prepare emergency means

### Patterns of Innovation
Common innovation patterns:
- **Removal**: Take away assumed necessary element
- **Addition**: Add unexpected element  
- **Multiplication**: Multiple identical/modified versions
- **Division**: Separate components in time/space
- **Task unification**: Existing resource does new job
- **Attribute dependency**: Make variables correlate

## Constraints-Based Creativity

### Restrictive Constraints
Limitations spark creativity.
- "Solve this with zero budget"
- "Design with only 3 components"
- "Explain in one sentence"
- Forces novel approaches

### Forced Fit
Given fixed elements, make it work.
- "You must use [X]. Now design around it."
- "These are your resources. No others."
- Drives creative use of available means

## Visualization & Metaphor

### Rich Pictures
Draw the problem system.
- Relationships, flows, tensions
- Non-technical stakeholders understand
- Reveals hidden connections

### Metaphor Exploration
Deep metaphor analysis.
- "Our organization is like a [X]"
- Extend metaphor: Implications?
- Switch metaphors: "What if instead it's like [Y]?"

### Mind Mapping
Radial, associative structure.
- Central concept
- Branch to related ideas
- Sub-branch from branches
- Visual connections, colors, images
