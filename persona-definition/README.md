# Persona Definition

> Create detailed user personas with demographics, goals, pain points, behaviors, and empathy maps.

## Overview

Products built without a clear picture of who they serve tend to serve no one well. Persona Definition provides the frameworks and templates needed to create research-backed user personas that drive product, design, and documentation decisions. It moves beyond demographic stereotypes to capture what actually matters: goals, pain points, behaviors, and context.

The skill supports three levels of persona fidelity -- proto-personas for quick team alignment, lean personas for agile and MVP development, and full personas for strategic product decisions. Each level is documented with its purpose, appropriate detail level, and when to choose it. The skill also includes an empathy map framework (Says/Thinks/Does/Feels) for deepening understanding of user psychology.

Within the SkillStack collection, Persona Definition is the user-understanding skill. It feeds directly into navigation design (personas validate that navigation matches user mental models), persona mapping (personas are the inputs to stakeholder analysis), and outcome orientation (outcomes are defined in terms of persona goals).

## Key Features

- Three persona fidelity levels with guidance on when to use each
- Five-component persona structure covering demographics through environmental context
- Empathy map quadrant framework for understanding user psychology

- Anti-pattern identification: aspirational personas, too many personas, demographic-only, static personas
- Documentation-aware personas that capture format preferences and detail level needs

## Usage Examples

**Create personas for a developer tool:**
```
Create 3 user personas for our CLI tool: a senior backend developer, a junior full-stack developer, and a DevOps engineer. For each, define their goals, pain points, tech savviness level, and what they need from our documentation.
```

**Build a proto-persona for quick alignment:**
```
We're starting a new project and need quick alignment on our target user. Create a proto-persona for a startup CTO who evaluates developer tools. Keep it lightweight -- just the essentials for sprint planning decisions.
```

**Create empathy maps:**
```
We have interview transcripts from 8 users. Synthesize them into empathy maps for our 2 primary personas. Fill in the Says/Thinks/Does/Feels quadrants with actual quotes and observed behaviors.
```

**Audit existing personas:**
```
Review our current persona set against the anti-patterns list. Check if any are aspirational rather than evidence-based, if we have too many (more than 5), if any are demographic-only without goals and behaviors, and if they've been updated in the last year.
```

**Define documentation personas:**
```
Create personas specifically focused on documentation needs. For each persona, specify their preferred content format (tutorials vs. reference vs. examples), detail level tolerance, and how they typically search for information.
```

## Quick Start

1. **Choose your fidelity level** -- Use proto-personas for early-stage alignment, lean personas for agile teams, or full personas for strategic decisions.

2. **Gather inputs** -- For proto-personas, use team knowledge. For lean and full personas, use user interviews, analytics, and support tickets.

3. **Fill in the template** -- Use the persona template in SKILL.md: name, quote, role, tech savviness, primary goal, avoidance goal, pain points, and documentation needs.

4. **Build empathy maps** -- For each persona, fill in the Says/Thinks/Does/Feels quadrants to deepen psychological understanding.

5. **Limit to 3-5 personas** -- More than 5 dilutes focus. If you have more, prioritize by business value.

6. **Keep personas alive** -- Schedule quarterly reviews. Update with new research. Retire personas that no longer represent your user base.
